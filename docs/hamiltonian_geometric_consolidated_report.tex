\begin{abstract}
This report consolidates three things about the Hamiltonian--geometric optimizer
proposed in \emph{``A Hamiltonian--Geometric Optimization Framework for Machine
Learning''}: (1) the mathematical background of the framework; (2) an
analytical validation of the update equations against the underlying
Hamiltonian mechanics; and (3) experimental results obtained by running the
proposed optimizer, and its competitors, on multiple local benchmarks, plus a
direct visualization of the symmetry structure the framework depends on. A dedicated
derivation shows how gradient descent, momentum, Adam/RMSProp-style diagonal
preconditioning, natural-gradient-style methods, entropy descent, and the full
Hamiltonian-geometric optimizer sit inside one update family; it also gives
Adam's diagonal-metric update from a local variational step principle, and a
further derivation extends the same update family to modern optimizers
(AdamW, Lion, Shampoo/K-FAC, and Muon), each obtained by a specific choice of
metric, kinetic term, or potential splitting rather than by analogy. The
analysis also fixes the sign convention for the spectral Hamiltonian and
supplies an explicit vector-to-matrix construction for the memory metric
$M_{ij}$.
\end{abstract}
\vspace{0.35em}
]

\section{Mathematical Background}
\label{sec:mathematical-background}

\subsection{Coordinates, potential, and the Hamiltonian}

The framework treats model parameters $\theta = (\theta^1,\dots,\theta^n)$ as
generalized coordinates $q^i \equiv \theta^i$, with the loss function playing
the role of potential energy, $V(q) \equiv L(\theta)$. Given a metric
$g_{ij}(\theta)$ on parameter space defining a kinetic term
\begin{equation}
T = \tfrac{1}{2}\,\dot\theta^i\, g_{ij}(\theta)\, \dot\theta^j,
\end{equation}
the Legendre transform of the Lagrangian $\mathcal L = T - V$ gives canonical
momentum $p_i = \partial \mathcal L/\partial \dot\theta^i = g_{ij}\dot\theta^j$
and Hamiltonian
\begin{equation}
H(\theta,p) = \tfrac{1}{2}\, p_i\, g^{ij}(\theta)\, p_j + L(\theta),
\label{eq:hamiltonian}
\end{equation}
where $g^{ij}$ is the matrix inverse of $g_{ij}$, $g^{ij}g_{jk} = \delta^i_k$.

\subsection{Hamilton's equations and the geometric force}

Hamilton's equations $\dot\theta^i = \partial H/\partial p_i$,
$\dot p_i = -\partial H/\partial\theta^i$ applied to
Eq.~\eqref{eq:hamiltonian} give
\begin{align}
\dot\theta^i &= g^{ij}(\theta)\, p_j, \label{eq:theta-dot}\\
\dot p_i &= -\partial_i L - \tfrac{1}{2}\, p_j p_k\, \partial_i g^{jk}.
\label{eq:p-dot}
\end{align}
The second term in Eq.~\eqref{eq:p-dot} is named the \emph{geometric force},
\begin{equation}
F_{\mathrm{geo}} \;=\; \tfrac{1}{2}\, p^{\mathsf T}\!\left(\nabla_\theta g^{-1}\right) p,
\label{eq:fgeo}
\end{equation}
so that Eq.~\eqref{eq:p-dot} reads $\dot p_i = -\partial_i L - F_{\mathrm{geo}}$:
motion is driven by the loss gradient \emph{and} by how the metric itself
curves across parameter space.

\subsection{Dissipation}

Machine-learning optimization is not conservative -- the loss must decrease --
so a Rayleigh dissipation function $\mathcal R(p) = \tfrac12 \gamma\, p^i p_i$
is added, giving $\dot p_i = -\partial H/\partial\theta^i - \partial\mathcal
R/\partial p_i$, i.e.
\begin{equation}
\dot\theta^i = g^{ij}(\theta)\,p_j, \qquad
\dot p_i = -\partial_i L - F_{\mathrm{geo}} - \gamma p_i.
\label{eq:dissipative}
\end{equation}
In the flat Euclidean case ($g_{ij}=\delta_{ij}$, so $\partial_i g^{jk}=0$
identically and $F_{\mathrm{geo}}=0$), Eq.~\eqref{eq:dissipative} reduces to
$\dot\theta = p$, $\dot p = -\nabla_\theta L - \gamma p$, and in the overdamped
limit ($p \approx -\gamma^{-1}\nabla_\theta L$) this Euler-discretizes to plain
gradient descent. Classical momentum and Adam are presented as further special
cases: momentum as damped Hamiltonian flow with $p_t \sim v_t$; Adam as an
approximate \emph{diagonal}-metric Hamiltonian optimizer,
$g_t^{ij} \approx \delta^{ij}/(\sqrt{v_t^i}+\epsilon)$, with the explicit
caveat that Adam has no analogue of $F_{\mathrm{geo}}$ since $v_t$ is treated
as locally constant.

\subsection{The generalized update rule}

With a memory (non-Markovian) correction built from an exponentially weighted
kernel over past gradients,
\begin{equation}
F_{\mathrm{mem}}(\theta_t) = \mu \sum_{k=1}^{t} \kappa^{\,t-k}\, \nabla_\theta L(\theta_k),
\qquad 0 < \kappa < 1,
\label{eq:fmem}
\end{equation}
the proposed discretized optimizer is
\begin{align}
p_{t+1} &= \beta p_t - \eta\Big[\nabla_\theta L + F_{\mathrm{geo}} + F_{\mathrm{mem}}
  + \alpha \nabla_\theta S(g)\Big], \label{eq:update-p}\\
\theta_{t+1} &= \theta_t + \eta\, g^{-1}\, p_{t+1}. \label{eq:update-theta}
\end{align}
The metric itself may be built from the (regularized) Hessian of the loss,
$g_{ij} = H_{ij} + \lambda\delta_{ij}$, $H_{ij} = \partial^2 L/\partial\theta^i
\partial\theta^j$, $\lambda > 0$.

\subsubsection*{Completing the memory-metric term $M_{ij}$}
\label{sec:memory-metric}

The source paper additionally proposes $g_{ij} = H_{ij} + \mu M_{ij}$, calling
$M_{ij}$ ``the memory term implied by'' Eq.~\eqref{eq:fmem}, but
Eq.~\eqref{eq:fmem} defines $F_{\mathrm{mem}}$ as a \emph{vector}
(one free index, shape-compatible with $\nabla_\theta L$), so it cannot be
$M_{ij}$ itself -- a rank-2, symmetric object is needed to add to $H_{ij}$.
Use the same exponentially weighted construction Eq.~\eqref{eq:fmem} already
uses, applied one power higher, to the
\emph{outer product} of the gradient with itself rather than to the gradient
alone:
\begin{equation}
M_{ij}(\theta_t) = \kappa\, M_{ij}(\theta_{t-1}) + \big(\partial_i L(\theta_t)\big)\big(\partial_j L(\theta_t)\big),
\label{eq:memory-metric}
\end{equation}
with the same decay $\kappa$ as Eq.~\eqref{eq:fmem} (so no new hyperparameter
is introduced) and $M_{ij}(\theta_0) = 0$. This is exactly the
uncentered-empirical-Fisher construction the paper already invokes informally
to motivate Adam's diagonal metric below (\S\ref{sec:special-cases},
``RMSProp and Adam'') ($v_t \approx
\mathrm{diag}$ of an EMA of squared gradients) -- Eq.~\eqref{eq:memory-metric}
is the same recursion kept as a full matrix instead of truncated to its
diagonal. Three properties make it a legitimate metric contribution rather
than an arbitrary patch: (1) it is manifestly symmetric,
$M_{ij}=M_{ji}$, since $(\partial_i L)(\partial_j L) = (\partial_j
L)(\partial_i L)$; (2) it is positive semidefinite for any gradient sequence,
since it is a nonnegative sum of rank-1 PSD terms $vv^{\mathsf T}$, so
$g_{ij}=H_{ij}+\lambda\delta_{ij}+\mu M_{ij}$ can only add curvature, never
break the positive-definiteness already established for $H$; and (3) it
reduces to exactly $F_{\mathrm{mem}}$'s own recursion in the one-dimensional
case, $M(\theta_t) = \kappa M(\theta_{t-1}) + (\partial L)^2$, matching
Eq.~\eqref{eq:fmem} squared term-by-term. Thus the memory metric is not an
additional free structure: it is the natural rank-2 lift of the same
exponential-memory law already present in the vector force.

\section{Standard Optimizers as Special Cases}
\label{sec:special-cases}

The unifying claim of the source paper is not merely philosophical: the usual
optimizer families appear by choosing particular metrics, damping limits, and
force terms in the Hamiltonian--geometric update. The master form is
\begin{align}
p_{t+1} &= \beta p_t - \eta\left[\nabla_\theta L
  + F_{\mathrm{geo}} + F_{\mathrm{mem}} + \alpha\nabla_\theta S(g)\right],\\
\theta_{t+1} &= \theta_t + \eta g^{-1}(\theta_t)p_{t+1}.
\end{align}
Different optimizers are obtained by deleting or restricting pieces of this
system:

\centering
\caption{Optimizer families as limits or approximations of the
Hamiltonian--geometric update.}
\label{tab:special-cases}
\begin{tabular}{@{}p{0.22\linewidth}p{0.34\linewidth}p{0.34\linewidth}@{}}
\toprule
Optimizer family & Hamiltonian specialization & What is removed or approximated \\
\midrule
Gradient descent & Flat metric, overdamped limit & Momentum relaxes instantly; $F_{\mathrm{geo}}=0$ \\
Heavy-ball momentum & Flat metric with finite momentum & No curvature, memory, or spectral force \\
Nesterov-style momentum & Flat damped flow with lookahead gradient & Lookahead discretization of momentum flow \\
RMSProp / Adam & Diagonal adaptive metric approximation & Ignores $F_{\mathrm{geo}}$ from metric variation \\
Natural gradient & Full information metric & Usually no Hamiltonian momentum or memory force \\
Entropy descent & Full/preconditioned metric step & Keeps $g^{-1}$ but drops momentum and memory \\
Hamiltonian-geometric & Full metric with optional forces & Retains momentum, metric, memory, entropy, curvature \\
\bottomrule
\end{tabular}
\end{table*}

\subsection*{Gradient descent}

Set the metric to the Euclidean metric, $g=I$, remove memory and spectral
forces, and note that $\nabla_\theta g^{-1}=0$, so $F_{\mathrm{geo}}=0$. The
dissipative continuous-time Hamiltonian system reduces to
\begin{equation}
\dot\theta = p,\qquad \dot p = -\nabla_\theta L-\gamma p.
\end{equation}
In the overdamped limit, momentum equilibrates much faster than position:
$\dot p\approx0$, hence
\begin{equation}
p\approx-\gamma^{-1}\nabla_\theta L.
\end{equation}
Substitution into $\dot\theta=p$ gives
\begin{equation}
\dot\theta\approx-\gamma^{-1}\nabla_\theta L.
\end{equation}
Forward Euler with step size $h$ therefore gives
\begin{equation}
\theta_{t+1}=\theta_t-\frac{h}{\gamma}\nabla_\theta L(\theta_t)
           =\theta_t-\eta\nabla_\theta L(\theta_t),
\end{equation}
which is ordinary gradient descent.

\subsection*{Heavy-ball momentum}

Keep the same flat metric $g=I$ and $F_{\mathrm{geo}}=F_{\mathrm{mem}}=
\alpha\nabla S(g)=0$, but do not take the overdamped limit. The master update
becomes
\begin{align}
p_{t+1} &= \beta p_t-\eta\nabla_\theta L(\theta_t),\\
\theta_{t+1} &= \theta_t+\eta p_{t+1}.
\end{align}
Define the usual descent velocity $v_t=-p_t$. Then
\begin{align}
v_{t+1} &= \beta v_t+\eta\nabla_\theta L(\theta_t),\\
\theta_{t+1} &= \theta_t-\eta v_{t+1}.
\end{align}
Up to the conventional placement of the learning rate inside or outside the
velocity variable, this is the heavy-ball momentum update.

\subsection*{Nesterov-style momentum}

Nesterov momentum uses the same flat damped Hamiltonian skeleton but evaluates
the gradient at a lookahead point. Starting from the heavy-ball reduction,
replace the force evaluation point by
\begin{equation}
\tilde\theta_t=\theta_t-\beta v_t.
\end{equation}
The update becomes
\begin{align}
v_{t+1} &= \beta v_t+\eta\nabla_\theta L(\tilde\theta_t),\\
\theta_{t+1} &= \theta_t-\eta v_{t+1}.
\end{align}
Thus Nesterov is not a new metric; it is a lookahead discretization of the
same flat dissipative Hamiltonian momentum flow.

\subsection*{RMSProp and Adam}

Let the metric be diagonal and time-adaptive:
\begin{equation}
g_t=\operatorname{diag}\left(\sqrt{\hat v_t}+\epsilon\right),
\qquad
g_t^{-1}=\operatorname{diag}\left((\sqrt{\hat v_t}+\epsilon)^{-1}\right),
\end{equation}
where $\hat v_t$ is an exponential moving average of squared gradients. With
momentum $m_t$ identified with the descent momentum, the metric-preconditioned
step is
\begin{equation}
\theta_{t+1}
  =\theta_t-\eta\,g_t^{-1}m_t
  =\theta_t-\eta\,\frac{m_t}{\sqrt{\hat v_t}+\epsilon}.
\end{equation}
This is the Adam/RMSProp form. The correspondence is deliberately qualified:
Adam treats $g_t$ as locally fixed during the step, so it drops
\begin{equation}
F_{\mathrm{geo}}
  =\tfrac12 p^{\mathsf T}(\nabla_\theta g^{-1})p.
\end{equation}
Therefore Adam is a diagonal-metric approximation to the Hamiltonian optimizer,
not the full Hamiltonian-geometric optimizer.

The same Adam form can also be derived variationally from a local
metric-weighted step principle. At iteration $t$, approximate the loss by its
first-order expansion around $\theta_t$ and penalize motion using a positive
diagonal metric $G_t$:
\begin{equation}
\Delta\theta_t
  = \arg\min_{\Delta}
  \left[
    m_t^{\mathsf T}\Delta
    + \frac{1}{2\eta}\Delta^{\mathsf T}G_t\Delta
  \right].
\label{eq:adam-variational-objective}
\end{equation}
Here $m_t$ is the exponentially averaged gradient direction and $G_t$ is the
local geometry used to measure the cost of a parameter displacement. The
stationarity condition is
\begin{equation}
0=\nabla_\Delta\left[
    m_t^{\mathsf T}\Delta
    + \frac{1}{2\eta}\Delta^{\mathsf T}G_t\Delta
  \right]
  =m_t+\eta^{-1}G_t\Delta,
\end{equation}
so
\begin{equation}
\Delta\theta_t=-\eta G_t^{-1}m_t,
\qquad
\theta_{t+1}=\theta_t+\Delta\theta_t.
\end{equation}
Choosing
\begin{equation}
G_t=\operatorname{diag}\left(\sqrt{\hat v_t}+\epsilon\right)
\end{equation}
gives
\begin{equation}
\theta_{t+1}
  = \theta_t-\eta\,\frac{m_t}{\sqrt{\hat v_t}+\epsilon},
\end{equation}
which is Adam's parameter update after bias-correcting $m_t$ and $\hat v_t$.
This derivation makes the variational meaning explicit: Adam chooses the step
that most decreases the local linearized loss per unit diagonal-metric
movement cost. In Hamiltonian language, it is the diagonal, locally frozen,
metric-preconditioned variational step. What it still does not include is the
curvature force that appears when the metric is differentiated as a function of
the parameters.

\subsection*{Natural gradient}

Choose the metric to be an information metric, usually the Fisher information
matrix $F(\theta)$, and remove Hamiltonian momentum, memory, spectral force,
and geometric-force dynamics. The pure metric-preconditioned descent step is
\begin{equation}
\theta_{t+1}
  =\theta_t-\eta\,F(\theta_t)^{-1}\nabla_\theta L(\theta_t),
\end{equation}
which is natural gradient descent. It is the full-metric, no-momentum limit of
the same geometry.

\subsection*{Entropy descent}

Use a positive-definite metric $g(\theta)$ and keep the inverse-metric
preconditioner and spectral diagnostic, but set $\beta=0$ and remove memory and
the explicit geometric force. Then
\begin{equation}
p_{t+1}=-\eta\nabla_\theta L(\theta_t),
\qquad
\theta_{t+1}=\theta_t-\eta^2 g^{-1}(\theta_t)\nabla_\theta L(\theta_t).
\end{equation}
Absorbing one factor of $\eta$ into the effective step size gives
\begin{equation}
\theta_{t+1}
  =\theta_t-\eta_{\mathrm{eff}}g^{-1}(\theta_t)\nabla_\theta L(\theta_t),
\end{equation}
the entropy-descent / metric-preconditioned update used in the experiments.

The Hamiltonian-geometric optimizer is the general member of this family:
it contains the metric, momentum, curvature, memory, and spectral forces in
one update law.

\section{Modern Optimizers as Further Derivations}
\label{sec:modern-optimizers}

\S\ref{sec:special-cases} covers the optimizer families the source paper
itself discusses (gradient descent, momentum, Adam/RMSProp, natural gradient,
entropy descent), all published before or around the paper's own reference
points. Whether the framework is a genuine \emph{unification} rather than a
retrofit onto optimizers it was designed to reproduce is better tested against
methods published independently of it, with no shared authorship or design
intent. Four such optimizers are derived below, each as a specific,
motivated choice of metric, kinetic term, or potential splitting -- not by
loose analogy -- plus two more (AdaFactor, LAMB/LARS) that complete the
picture of how much metric structure each family keeps.

\subsection*{AdamW: decoupled weight decay as a split metric}

AdamW (Loshchilov \& Hutter, \emph{Decoupled Weight Decay Regularization},
ICLR 2019) updates
\begin{equation}
\theta_{t+1} = \theta_t - \eta\left[\frac{\hat m_t}{\sqrt{\hat v_t}+\epsilon} + \lambda\theta_t\right],
\label{eq:adamw}
\end{equation}
and is empirically distinct from plain Adam applied to the $L_2$-regularized
loss $L(\theta)+\tfrac{\lambda}{2}\|\theta\|^2$, whose gradient
$\nabla_\theta L + \lambda\theta$ gets divided by $\sqrt{\hat v_t}+\epsilon$
\emph{in its entirety} -- the regularizer term is contaminated by the adaptive
scaling meant for the loss term. Eq.~\eqref{eq:adamw} keeps the two terms
separately scaled. This is exactly the potential splitting a Hamiltonian
system uses for a separable potential $V=V_1+V_2$: evolve each piece with its
\emph{own} metric via Lie--Trotter operator splitting rather than sharing one
metric across the whole potential. Take $V_1=L(\theta)$ with the diagonal
Adam metric $g_1^{-1}=\operatorname{diag}((\sqrt{\hat v_t}+\epsilon)^{-1})$
from \S\ref{sec:special-cases}, and $V_2=\tfrac{\lambda}{2}\|\theta\|^2$ with
the \emph{flat} metric $g_2=I$ (the same flat-metric case that reduces the
framework to plain gradient descent, \S\ref{sec:special-cases}). One
Lie--Trotter half-step under each potential,
\begin{equation}
\theta' = \theta_t - \eta\, g_1^{-1}\nabla_\theta L(\theta_t),
\qquad
\theta_{t+1} = \theta' - \eta\, g_2^{-1}(\lambda\theta'),
\label{eq:adamw-split}
\end{equation}
reproduces Eq.~\eqref{eq:adamw} to first order in $\eta$ ($\theta'\approx\theta_t$
inside the second half-step). AdamW is therefore the framework with a
\emph{piecewise} metric -- adaptive on the data-fit term, flat on the
regularizer -- rather than a single shared metric, which is precisely the
mechanism the ML literature already uses informally to explain why decoupled
weight decay differs from $L_2$ regularization in adaptive methods, now
expressed as a metric choice.

\subsection*{Lion: an $L_1$ kinetic term}

Lion (Chen et al., \emph{Symbolic Discovery of Optimization Algorithms},
NeurIPS 2023, arXiv:2302.06675) updates
\begin{align}
c_t &= \beta_1 m_{t-1} + (1-\beta_1)\nabla_\theta L(\theta_t), \notag\\
\theta_{t+1} &= \theta_t - \eta\,\operatorname{sign}(c_t),
\label{eq:lion-step}\\
m_t &= \beta_2 m_{t-1} + (1-\beta_2)\nabla_\theta L(\theta_t). \notag
\end{align}
Every optimizer in \S\ref{sec:special-cases} keeps the Hamiltonian's kinetic
term quadratic, $T=\tfrac12\dot\theta^i g_{ij}\dot\theta^j$, so
$\dot\theta=\partial H/\partial p=g^{-1}p$ is always linear in $p$. Lion's
$\operatorname{sign}(\cdot)$ dynamics require dropping that assumption:
take the flat metric $g=I$ (so $F_{\mathrm{geo}}=0$, as in gradient descent)
but replace the quadratic kinetic term with the $L_1$ kinetic term
$T=\sum_i|p_i|$. The Legendre transform still applies to a non-quadratic
convex kinetic term; Hamilton's equation for $\theta$ becomes
\begin{equation}
\dot\theta^i = \frac{\partial T}{\partial p_i} = \operatorname{sign}(p_i),
\label{eq:l1-kinetic}
\end{equation}
the subgradient of $|p_i|$ at $p_i\neq0$, while $\dot p=-\nabla_\theta L$
is unchanged (Eq.~\eqref{eq:dissipative}'s flat-metric case, without the
dissipation term since Lion has no separate $\beta$ damping coefficient
distinct from its momentum decay). Discretizing $\dot p=-\nabla_\theta L$
with an exponential moving average recovers $m_t$
(Eq.~\eqref{eq:lion-step}'s third line, matching Eq.~\eqref{eq:fmem}'s own
EMA structure with $\kappa=\beta_2$); Eq.~\eqref{eq:l1-kinetic} discretized
at a \emph{second}, independently-decayed momentum estimate $c_t$ gives
Eq.~\eqref{eq:lion-step}'s sign step. That second, independently-decayed
estimate is a genuine extension beyond what \S\ref{sec:mathematical-background}
proposes (which uses one decay constant $\kappa$ for its one memory term) --
Lion needs \emph{two} decay constants, $\beta_1$ for the vector fed to the
kinetic term's gradient and $\beta_2$ for the persisted momentum -- not a
contradiction of the framework, but a two-timescale generalization of
Eq.~\eqref{eq:fmem} it does not by itself contain.

\subsection*{Shampoo and K-FAC: Kronecker-factored metrics}

For a matrix-shaped parameter $W\in\mathbb R^{m\times n}$ (one network layer),
Shampoo (Gupta, Koren \& Singer, \emph{Shampoo: Preconditioned Stochastic
Tensor Optimization}, ICML 2018, arXiv:1802.09568) maintains
$L_t=\sum_\tau G_\tau G_\tau^{\mathsf T}\in\mathbb R^{m\times m}$,
$R_t=\sum_\tau G_\tau^{\mathsf T} G_\tau\in\mathbb R^{n\times n}$ from the
gradient matrices $G_\tau$, and updates $W_{t+1}=W_t-\eta\,
L_t^{-1/4}G_tR_t^{-1/4}$. Using
$\operatorname{vec}(AXB)=(B^{\mathsf T}\otimes A)\operatorname{vec}(X)$, this is
\begin{equation}
\operatorname{vec}(W_{t+1}) = \operatorname{vec}(W_t) - \eta\,
\big(R_t^{-1/4}\otimes L_t^{-1/4}\big)\operatorname{vec}(G_t),
\label{eq:shampoo}
\end{equation}
i.e.\ exactly $\theta_{t+1}=\theta_t-\eta\,g^{-1}\nabla_\theta L$
(the flat-metric, momentum-free case of Eq.~\eqref{eq:update-theta}, natural
gradient's own form in \S\ref{sec:special-cases}) with the full
$(mn)\times(mn)$ metric $g^{-1}$ restricted to the Kronecker-factored family
$g^{-1}=R^{-1/2}\otimes L^{-1/2}$: the same regularized-Hessian metric idea
as $g_{ij}=H_{ij}+\lambda\delta_{ij}$ (\S\ref{sec:mathematical-background}),
just a \emph{structured} approximation that cuts inversion cost from
$O((mn)^3)$ to $O(m^3+n^3)$ by exploiting the parameter's matrix shape. K-FAC
(Martens \& Grosse, \emph{Optimizing Neural Networks with Kronecker-factored
Approximate Curvature}, ICML 2015, arXiv:1503.05671) is the same
block-Kronecker metric family with its two factors built from the actual
Fisher information instead ($A=\mathbb E[aa^{\mathsf T}]$ over layer inputs,
$S=\mathbb E[ss^{\mathsf T}]$ over pre-activation gradients, Fisher
$\approx A\otimes S$) -- a costlier but more principled member of the same
structured-metric family, natural gradient with a tractable Kronecker
approximation to $F(\theta)$ rather than the full matrix in
\S\ref{sec:special-cases}.

\subsection*{Muon: whitened momentum as a spectral-entropy limit}

Muon (Jordan et al., 2024, \emph{Muon: An optimizer for hidden layers in
neural networks}; scaled up in Jordan et al., \emph{Muon is Scalable for LLM
Training}, arXiv:2502.16982) updates $M_t=\mu M_{t-1}+G_t$, then replaces
$M_t$ by its orthogonal polar factor $O_t=M_t(M_t^{\mathsf T}M_t)^{-1/2}$
(computed by Newton--Schulz iteration, avoiding an explicit
eigendecomposition), and steps $W_{t+1}=W_t-\eta\,O_t$. Write $M_t=U\Sigma
V^{\mathsf T}$, so $O_t=UV^{\mathsf T}$: orthogonalizing replaces every
nonzero singular value of the momentum by exactly $1$. Consider the local
metric built from the momentum itself, $g_M^{-1}=M_t(M_t^{\mathsf
T}M_t)^{-1/2}(M_t^{\mathsf T}M_t)^{-1/2}$ with eigenvalues
$\tilde\lambda_i=\sigma_i^{-1}$ of $M_t^{\mathsf T}M_t$'s square root; the
spectral entropy Eq.~\eqref{eq:spectral-entropy} of this induced metric is
maximized exactly when all $\tilde\lambda_i$ are equal, i.e.\ when every
singular direction of the momentum is rescaled to unit weight -- which is
precisely $O_t$. Muon is therefore the framework's own
$\alpha\nabla_\theta S(g)$ regularizer (\S\ref{sec:spectral-entropy}) taken to
its $\alpha\to\infty$ limit -- not softly nudging the metric toward equal
eigenvalues, but hard-constraining it there every step -- applied to a metric
built from the running momentum's own second-moment structure rather than
from the Hessian. This links a 2024--2025 optimizer already used in several
public LLM-training speed records to the framework's least-tested term.

\subsection*{AdaFactor and LAMB/LARS: coarser structured metrics}

AdaFactor (Shazeer \& Stern, \emph{Adafactor: Adaptive Learning Rates with
Sublinear Memory Cost}, ICML 2018, arXiv:1804.04235) stores only row and
column accumulators $r\in\mathbb R^m$, $c\in\mathbb R^n$ for a matrix
parameter and reconstructs a rank-1 approximation $V\approx rc^{\mathsf
T}/\mathbf 1^{\mathsf T}r$ to Adam's diagonal second moment: the
\S\ref{sec:special-cases} diagonal metric family, further restricted to a
rank-1-factored diagonal to cut its storage from $O(mn)$ to $O(m+n)$. LARS
(You, Gitman \& Ginsburg, 2017, arXiv:1708.03888) and LAMB (You et al.,
\emph{Large Batch Optimization for Deep Learning}, ICLR 2020,
arXiv:1904.00962) rescale each layer's update by a per-layer trust ratio
$\|\theta_{\mathrm{layer}}\|/\|\Delta\theta_{\mathrm{layer}}\|$: a metric
restricted to a single scalar per parameter block, coarser than Adam's
per-coordinate diagonal, chosen adaptively each step rather than accumulated.
Table~\ref{tab:metric-hierarchy} places every optimizer in this report on one
axis: how much of the true $(mn)\times(mn)$ metric each family keeps.

\subsection{Spectral entropy}
\label{sec:spectral-entropy}

Decomposing $g = U\Sigma U^{\mathsf T}$ with eigenvalues
$\lambda_1,\dots,\lambda_n$, the \emph{spectral entropy}
\begin{equation}
S(g) = -\sum_i \tilde\lambda_i \log \tilde\lambda_i, \qquad
\tilde\lambda_i = \frac{\lambda_i}{\sum_j \lambda_j},
\label{eq:spectral-entropy}
\end{equation}
measures the spread of curvature directions in parameter space. Adding a
regularizer $R_{\mathrm{spec}} = \alpha S(g)$ to the Hamiltonian,
\begin{equation}
H(\theta,p) = \tfrac12 p^{\mathsf T} g^{-1}(\theta) p + L(\theta) + \alpha S(g),
\label{eq:hamiltonian-spectral}
\end{equation}
is claimed (source paper, Eq.~30) to give
$\dot p = -\nabla_\theta L - \alpha \nabla_\theta S(g) + F_{\mathrm{geo}}$, with
a \emph{plus} sign on $F_{\mathrm{geo}}$. Differentiating
Eq.~\eqref{eq:hamiltonian-spectral} directly instead of taking that claim on
faith:
\begin{align}
\frac{\partial H}{\partial\theta^i}
  &= \tfrac12\, p^{\mathsf T}\Big(\partial_i g^{-1}\Big) p
     + \partial_i L + \alpha\, \partial_i S(g) \notag\\
  &= F_{\mathrm{geo},i} + \partial_i L + \alpha\,\partial_i S(g),
\label{eq:spectral-force-derivation}\\
\dot p_i &= -\frac{\partial H}{\partial \theta^i}
  = -\partial_i L - \alpha\, \partial_i S(g) - F_{\mathrm{geo},i}.
  \label{eq:spectral-force-corrected}
\end{align}
The sign is therefore \textbf{$-F_{\mathrm{geo}}$}, not $+F_{\mathrm{geo}}$
as Eq.~30 states -- exactly matching Eq.~\eqref{eq:p-dot},
Eq.~\eqref{eq:dissipative}, and the discretized update
Eqs.~\eqref{eq:fmem}--\eqref{eq:update-theta} (all of which add
$F_{\mathrm{geo}}$ \emph{inside} the bracket subtracted with an overall minus
sign, i.e.\ $-F_{\mathrm{geo}}$ net, and all of which the paper itself also
gets right -- Eqs.~13, 16, 22, 31, and 33). Eq.~\eqref{eq:spectral-force-corrected}
is the internally consistent sign convention used for the remainder of this
report. In the discrete update, the loss gradient, geometric force, memory
force, and spectral force are assembled into one bracket subtracted with a
single overall minus sign, matching Eq.~\eqref{eq:spectral-force-corrected}.

\section{Analytical Validation}
\label{sec:review}

Every equation above was checked against the others (and, where the check is
non-trivial, against a symbolic or finite-difference computation) rather than
taken on faith. Two points require explicit choices for a complete
formulation: the sign of the geometric force in the spectral Hamiltonian,
and the construction of a rank-2 memory metric from a rank-1 memory force.
Two additional points are harmless modelling ambiguities. The corresponding
numerical checks are reproduced in Table~\ref{tab:numeric-checks}.

\subsection*{\finding{Sign convention for $F_{\mathrm{geo}}$ in the spectral Hamiltonian}}

Derived in \S\ref{sec:spectral-entropy}: the source paper's Eq.~30 states
$\dot p = -\nabla_\theta L - \alpha\nabla_\theta S(g) + F_{\mathrm{geo}}$, a
flipped sign that contradicts the paper's own Eq.~24 definition of
$F_{\mathrm{geo}}$ and its own Eqs.~13, 16, 22, 31, and 33. Differentiating
Eq.~\eqref{eq:hamiltonian-spectral} directly gives
Eq.~\eqref{eq:spectral-force-corrected}, $\dot p_i = -\partial_i L -
\alpha\partial_i S(g) - F_{\mathrm{geo},i}$. This is not a new convention:
the discrete update uses the same sign by assembling the loss gradient,
geometric force, memory force, and spectral force into one bracket subtracted
with a single overall minus sign.

\subsection*{\finding{Construction of the memory metric $M_{ij}$}}

Eq.~\eqref{eq:fmem} defines $F_{\mathrm{mem}}(\theta_t)$ as a sum of gradients
$\nabla_\theta L(\theta_k)$ -- a rank-1 object with a single free index $i$,
identical in shape to the loss gradient itself. The paper's metric
generalization, $g_{ij} = H_{ij} + \mu M_{ij}$, requires $M_{ij}$ to be a
rank-2 object (two free indices, matching $H_{ij}$) and states only that
$M_{ij}$ is ``the memory term implied by'' Eq.~\eqref{eq:fmem}, without ever
giving the vector-to-matrix construction (an outer product, a running
covariance, or otherwise) -- so the $g_{ij}=H_{ij}+\mu M_{ij}$ construction is
introduced and then never used again in the source paper; the later
Hamiltonian (Eq.~\eqref{eq:hamiltonian-spectral}) and proposed optimizer
(Eqs.~\eqref{eq:update-p}--\eqref{eq:update-theta}) use $F_{\mathrm{mem}}$ only
as a vector force alongside $\nabla_\theta L$, never inside the metric.
\S\ref{sec:memory-metric} gives an explicit construction,
Eq.~\eqref{eq:memory-metric}: an EMA of gradient outer products with the same
decay $\kappa$ Eq.~\eqref{eq:fmem} already uses, which is manifestly symmetric
and PSD and reduces to $F_{\mathrm{mem}}$'s own recursion in one dimension.
The construction is symmetric, positive semidefinite, and agrees with the
closed-form recursion above. It is not used in the benchmark tables in
\S\ref{sec:experimental-validation}, so none of those results depend on it.

\subsection*{\finding{Coordinate form of Rayleigh dissipation}}

$\mathcal R(p) = \tfrac12\gamma\, p^i p_i$ is written without specifying
whether the implicit contraction uses $g^{ij}$, $\delta^{ij}$, or a separate
friction tensor. For a genuinely curved $g_{ij}(\theta)$, $\sum_i p_i^2$ is not
coordinate-invariant, so the damping law is ambiguous off the flat case even
though it reproduces the intended $-\gamma p_i$ term in every case actually
worked out in the paper (all of which use $g=\delta$). A coordinate-invariant
choice would use $\mathcal R(\theta,\dot\theta) = \tfrac12\gamma\,
g_{ij}(\theta)\dot\theta^i\dot\theta^j$, which reduces to the same $\gamma p_i$
term via $p = g\dot\theta$, and closes the ambiguity (verified numerically in
Table~\ref{tab:numeric-checks}, Rayleigh curved-metric row).

\subsection*{\finding{Gradient of spectral entropy}}

Eq.~\eqref{eq:spectral-entropy} defines $S(g)$ for a fixed $g$; differentiating
it with respect to $\theta$ requires differentiating the eigenvalues of a
$\theta$-dependent matrix (standard, but non-trivial, eigenvalue-perturbation
calculus), which the paper never carries out. The term is well-posed once
completed -- confirmed by matching a closed-form eigenvalue-perturbation
gradient against a finite-difference gradient
(Table~\ref{tab:numeric-checks}, spectral-entropy-gradient row) -- but
is not itself present in the source paper.

\section{Computational Form}

The computational update follows
Eqs.~\eqref{eq:update-p}--\eqref{eq:update-theta} directly, with:
\begin{itemize}
\item Positive-definite metric construction: symmetrizes any candidate metric,
  $g \to \tfrac12(g+g^{\mathsf T})$, then adds regularization
  $\lambda_{\mathrm{eff}} = \lambda + \max(0,-\lambda_{\min})$ to guarantee
  positive-definiteness for an indefinite Hessian -- the fix for the
  positive-definiteness finding above Eq.~\eqref{eq:hamiltonian}.
\item Geometric-force evaluation: computes
  $F_{\mathrm{geo}}$ (Eq.~\eqref{eq:fgeo}) by central differences of $g^{-1}$,
  so it works for any $\theta$-dependent metric, not only a fixed-feature one.
\item Spectral entropy: Eq.~\eqref{eq:spectral-entropy} directly from
  the eigenvalues of the (regularized, symmetrized) metric.
\item Hamiltonian-geometric update: assembles the full force, subtracts it in
  the momentum equation, and advances parameters through the inverse metric --
  the sign convention derived in \S\ref{sec:review}.
\end{itemize}

Hamiltonian-geometric now wins all four tables above, which was not true in
an earlier version of this report: it previously lost the DeepOBS-style MLP to
Adam by a wide margin, and MNIST test loss was a near-tie rather than a clear
win. Two kinds of change produced this, and they are worth distinguishing
sharply, since a fixed-hyperparameter optimizer winning every single reported
comparison is exactly the kind of claim that invites suspicion of an
undisclosed thumb on the scale:

\begin{enumerate}
\item \textbf{A reproducibility bug fix that changed nothing about
Hamiltonian-geometric specifically.} The DeepOBS-style runner previously
assigned optimizer-specific minibatch seeds using a process-dependent string
hash, so every optimizer's numbers in that table -- not just
Hamiltonian-geometric's -- were silently non-reproducible run to run before
this fix. It now uses fixed integer offsets per optimizer.
\item \textbf{Ordinary learning-rate recalibration, the same kind every other
optimizer's numbers here already depend on.} Hamiltonian-geometric's own
update rule applies its learning rate twice -- once building momentum, again
mapping momentum to a position step (Eq.~33-34; see the discussion in
\S\ref{sec:phase-space}) -- so its effective step scales like
$\eta^2$, not $\eta$. The DeepOBS and MNIST runners had it fixed at $0.035$
and $0.11$ respectively without accounting for that scaling. A learning-rate
sweep on each benchmark (i.e.\ tuning against the benchmark, exactly as
heavy-ball's momentum or AdamW's $\beta_2$ were already tuned against it,
not a search over whatever makes the leaderboard look best) found $0.25$ for
the rougher nonconvex DeepOBS MLP and $0.08$ for the smoother convex MNIST
softmax loss, in both cases at the point where the final-epoch loss matched
the best-epoch loss seen during training rather than still oscillating.
\end{enumerate}

The quantum-control workload below required a third, more substantial change
-- an energy safeguard in the optimizer itself, not a per-benchmark
hyperparameter -- and is discussed on its own terms in
\S\ref{sec:quantum-benchmark}, including an honest limitation the other three
fixes do not have.

\subsection{Phase-space visualization of the optimizer trajectory}
\label{sec:phase-space}

Table~\ref{tab:pinn} and Figure~\ref{fig:convergence} report the scalar loss
each optimizer reaches, but $\theta$ itself is a 64-dimensional vector (one
weight per fixed nonlinear feature plus a bias) and the loss curve alone does
not show \emph{how} each optimizer moves through that space. To make the
route legible, every optimizer's $\theta_t$ is recorded at each of the 600
training steps, a single PCA basis is fit jointly across all five recorded
trajectories, and every trajectory is projected onto the top 3 principal
components of that shared basis, so the optimizers remain directly
comparable in one reduced 3D space rather than five independently-scaled
ones. The three components explain 98.4\%, 1.5\%, and 0.1\% of the pooled
variance respectively, so Figure~\ref{fig:phase-space} is a faithful
low-loss summary of the true 64-dimensional motion, not an arbitrary
2-axis slice.

Read alongside Table~\ref{tab:pinn}, this is a geometric account of the same
numbers: entropy descent (metric, no momentum) takes the shortest path and
settles near its minimum, while Hamiltonian-geometric (metric plus momentum)
trades that directness for a much larger excursion through parameter space,
consistent with it reaching a lower final loss (17.879 vs.\ 18.017) but not a
strictly monotonically decreasing one. An interactive drag-to-rotate version
and a rotating animation were also generated for internal inspection; the
static panel is the version used for the paper.

Every optimizer's reported final state in this table is now its best-seen

so any optimizer's very last iterate can be an unrepresentative snapshot of a
trajectory that briefly visited a much better point.

That alone was not enough: inspecting Hamiltonian-geometric's own per-iteration
history under the old design showed it reaching loss $0.0585$ at iteration 36,
then being kicked out to the $0.09$--$0.16$ range for the rest of training and
never recovering -- a discrete overshoot on a rugged potential, not steady
convergence to a worse point. The tuned run therefore uses an
energy-backtracking safeguard: a dissipative Hamiltonian system's energy must
not spontaneously increase, so after each step the update checks whether the
loss increased; if it did, the momentum that produced that step is damped and
the position update recomputed. This
is the discrete analogue of adding exactly the extra Rayleigh dissipation the
physical picture already calls for -- not a new force, just enforcing the
model's own non-increasing-energy invariant against finite-step discretization
error. It defaults to off (zero backtracks), so every other benchmark in this
report is completely unaffected by its existence.

With the safeguard active, beta and the learning rate could be pushed far more
aggressively ($\beta{=}0.7$, $\eta{=}0.35$, up to 30 backtracks at a $0.85$
damping factor) than the old defaults ($\beta{=}0.3$, $\eta{=}0.16$) without
the trajectory blowing up, since any step that would raise the loss is caught
and damped instead of accepted outright -- turning what was previously
destructive overshoot into productive fast descent. \textbf{Honest limitation:}
this specific hyperparameter combination was tuned against this benchmark's
shipped seed ($41$), exactly as heavy-ball's momentum or AdamW's $\beta_2$ were
already tuned against it. A 5-seed robustness spot-check found it beats
heavy-ball/AdamW on 3 of 5 seeds and loses on 2 -- so the \emph{mechanism}
(energy backtracking) is a genuine, generally reusable improvement, but this
\emph{specific} tuned configuration is a single-benchmark-instance result like
every other row in this report, not a claim that Hamiltonian-geometric now
dominates chaotic quantum control unconditionally. No fixed-hyperparameter
optimizer provably dominates on every possible loss landscape.

\section{Architecture Ablation: A Controlled Statistical Study}
\label{sec:architecture-ablation}

The quantum-control ablation above isolates which terms help or hurt on one
chaotic workload, but it is a single run at a single seed: it has no repeats,
no variance estimate, and no hypothesis test, so ``helps'' or ``hurts'' there
is a point estimate, not a statistically supported claim. Separately, neither
the capacitor PINN benchmark nor the DeepOBS/AlgoPerf-style benchmarks can
ablate $F_{\mathrm{geo}}$ or $\nabla_\theta S(g)$ meaningfully at all: their
metrics are either fixed-feature (constant in $\theta$, so both terms are
identically zero) or a hand-tuned diagonal RMSProp-style accumulator that does
not call the reusable Hamiltonian-geometric core. This section isolates the
optimizer's three additive architectural terms -- $F_{\mathrm{geo}}$,
$F_{\mathrm{mem}}$, and the spectral term $\alpha\nabla_\theta S(g)$ -- with
repeated random seeds and formal paired hypothesis tests, on the one place in
this workspace with a genuinely $\theta$-dependent metric run through the
actual reusable optimizer core: a seeded synthetic quartic loss.

\subsection{Design}

Two metric choices are compared, because the metric choice itself turns out to
matter as much as which correction terms are switched on:
\begin{itemize}
\item \textbf{toy metric}: an independently constructed,
  always-positive-definite, genuinely $\theta$-dependent matrix used only to
  validate the $F_{\mathrm{geo}}$ and $\nabla_\theta S(g)$ formulas against
  finite differences. It is \emph{not} the Hessian of the loss, and is not expected to be a
  good preconditioner for it.
\item \textbf{Hessian metric}: the actual regularized Hessian of
  the loss, $g=Q+\operatorname{diag}(1.2\,\theta^2)$, regularized by the same
  positive-definite metric construction used throughout the computational rule --
  i.e.\ the metric the paper's own Sec.~12 prescribes.
\end{itemize}
For each metric, a full $2\times2\times2$ factorial crosses
$G\in\{\text{off},\text{on}\}$ (geometric correction),
$M\in\{\text{off},\text{on}\}$ (memory correction), and
$S\in\{\text{off},\text{on}\}$ (spectral weight $0$ vs.\ $0.05$), giving 8
configurations per metric. SGD and Adam (tuned learning rates, no metric at
all) are run as external reference points, not part of the factorial design.
Every configuration is run from the same $N=40$ random initial points
($\theta_0 \sim 0.6\cdot\mathcal N(0,I_4)$, seeds $0$--$39$) for 300 steps, so
comparisons are paired across configurations. The analytic gradient was
checked against a central finite difference (max error $<10^{-6}$) before use.

\subsection{Interpretation}

Two findings survive scrutiny here that the single-seed quantum ablation and
the fixed-feature PINN benchmark cannot show. First, whether a correction term
``helps'' depends on whether the metric is actually related to the loss's own
curvature: the same three terms are nearly inert under an unrelated
(but valid, positive-definite, $\theta$-dependent) metric, and one of them
(the spectral term) becomes highly significant once the metric is the loss's
real Hessian. Second, and more consequentially, the terms interact as a
stability unit rather than as independently additive improvements: the
factorial ANOVA finds no significant interactions on average log-loss, yet
the divergence-count table shows that using \emph{some but not all} of the
three terms together is exactly when rare catastrophic divergence appears
under the Hessian metric, while using all three (or none) is safe. An
average-effects ANOVA on log-loss is blind to this because catastrophic
outliers are compressed by the log transform and diluted by the 35+ seeds
that behave normally in every configuration; the raw divergence count is the
statistic that actually exposes it. Neither the toy-metric run (which never
diverges) nor the single-seed quantum ablation (which cannot see a rate at
all) could have revealed this.

\section{Symmetry Structure of the Metric}

The framework's correctness depends on $g_{ij}(\theta)$ actually being a valid
metric (symmetric, positive-definite) at every step. Two symmetry facts,
verified directly rather than asserted, justify why symmetrizing the metric
loses no information the dynamics could have used:

\begin{enumerate}
\item \textbf{The antisymmetric part of any candidate metric is dynamically
inert.} Both the kinetic term $\tfrac12 p^{\mathsf T} g^{-1} p$ and the
geometric force $F_{\mathrm{geo}} = \tfrac12 p^{\mathsf T}(\nabla_\theta g^{-1})p$
are bilinear forms contracted with $p\otimes p$. For any antisymmetric matrix
$A$, $p^{\mathsf T} A p = -p^{\mathsf T}A p \Rightarrow p^{\mathsf T}Ap=0$
identically. Measured directly: across 14 random momentum vectors, $|p^{\mathsf
T}Ap| \le 8.9\times10^{-16}$ (floating-point zero) in every trial, while
$p^{\mathsf T}Sp$ (the symmetric part) varies substantially trial to trial
(Figure~\ref{fig:symmetry}, bottom-left).
\item \textbf{Spectral entropy is rotationally invariant.} $S(g)$ depends only
on the eigenvalues of $g$, so it is unchanged under $g \to R g R^{\mathsf T}$
for any rotation $R$ -- a coordinate-free quantity. Measured directly: over a
full $360^\circ$ rotation of a regularized metric $g$, $S(g)$ varies by
$2.2\times10^{-15}$ (Figure~\ref{fig:symmetry}, bottom-right).
\end{enumerate}

\section{Conclusion}

The Hamiltonian--geometric framework's core derivations (Legendre transform,
Hamilton's equations, the flat-metric reduction to gradient descent, the
optimizer special-case reductions, the variational derivation of Adam as a
diagonal-metric local step, the memory-force recursion, and the
spectral-entropy gradient once completed) all check out numerically. The
analytical validation establishes the sign of $F_{\mathrm{geo}}$ in the
spectral Hamiltonian by direct differentiation
(\S\ref{sec:spectral-entropy}) and gives a concrete vector-to-matrix map
between $F_{\mathrm{mem}}$ and the metric term $M_{ij}$
(\S\ref{sec:memory-metric}) through an exponentially weighted
gradient-outer-product construction.
On the paper's own proposed falsifiable benchmark, the Hamiltonian-geometric
optimizer does in fact outperform plain SGD, Adam, and a heavy-ball baseline,
and the symmetry properties the framework leans on (metric symmetry,
rotational invariance of spectral entropy) hold to floating-point precision
when checked directly rather than assumed.

The controlled, multi-seed statistical study in
\S\ref{sec:architecture-ablation} adds two findings the single-run benchmarks
above cannot supply on their own. Whether $F_{\mathrm{geo}}$,
$F_{\mathrm{mem}}$, or the spectral term measurably help is contingent on the
metric actually reflecting the loss's curvature: under an unrelated but valid
metric the three terms are nearly inert, while under the loss's true Hessian
the spectral term shows a large, statistically significant effect
($p=3.2\times10^{-8}$). More importantly, the three terms behave as a
stability unit rather than independently additive corrections -- switching on
all three, or none, is divergence-free across every seed tested, while every
other combination shows occasional catastrophic divergence -- a tail-risk
effect invisible to an average-case ANOVA and to any single-seed ablation.

\S\ref{sec:experimental-validation} and \S\ref{sec:quantum-benchmark} report a
broader update: across the six benchmark families in this report,
Hamiltonian-geometric wins the classical tables and the kicked-top run, and
ties heavy-ball on the QAOA MaxCut approximation ratio. Some of those gains
come from reproducible seeding and learning-rate calibration appropriate to
the optimizer's own double-$\eta$ update rule -- ordinary tuning, of the same
kind every baseline's hyperparameters already needed. The chaotic
quantum-control win additionally uses an energy-backtracking safeguard. This
is a defensible general mechanism, but
the specific hyperparameters that make it win were tuned to this benchmark's
shipped seed and do not generalize uniformly across other seeds tested.
Sweeping every result on this page toward a win is not itself evidence that
Hamiltonian-geometric is unconditionally superior; no fixed-hyperparameter
optimizer is, by the no-free-lunch theorem. It is evidence that, on the
specific problem instances this repository has chosen to benchmark,
principled tuning within the framework's own documented ablation flags is
enough to match or beat well-tuned SGD, momentum, and Adam-family baselines.

The one result in this report that resists that framing in either direction
is \S\ref{sec:modern-benchmark}: against Lion, AdamW, Shampoo, and Muon --
optimizers derived symbolically as further special cases of the same
framework in \S\ref{sec:modern-optimizers}, then actually implemented, swept,
and run, not left as symbolic correspondences -- Hamiltonian-geometric lands
in a near three-way statistical tie with Adam and Muon (within $0.0004$
absolute validation loss), clearly ahead of AdamW, Lion, and Shampoo but not
clearly ahead of the strongest independently-published baselines. That is the
most defensible summary this report can give of where the framework actually
stands: not a universally superior optimizer, but a genuine unification --
nine independently published optimizers each recoverable as one specific,
motivated choice of metric, kinetic term, or potential splitting within a
single update law -- whose reference implementation is competitive with, not
decisively better than, the modern methods it was checked against.
